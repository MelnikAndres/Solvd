Conclusion


Critical Length

After analyzing the data, we can see there is no exact critical length from which bubble sort is always slower than the other algorithms, because ordered arrays will always be faster on bubble sort. However, we can notice that around length 80 cases were bubble sort is faster become less common.
The closer the array is to being ordered, the faster bubble sort will be. The closer the array is to being reversed, the slower bubble sort will be.


Complexity

Merge Sort and Quick Sort are generally faster than Bubble Sort, because they have a better complexity, which means they will eventually do less operations for each element when the array grows. Although, Quick Sort is not always faster than Merge Sort, because it has a worst case scenario, In-Place Quick Sort is the fastest algorithm overall, because swaping elements is usually faster than rearranging them.
